loan.json Schema

The uploaded file must be valid JSON with the following structure:

{ "total": 109000, // required — original loan amount (€) "interest_rate": 3.97, // optional — global annual interest rate (%); used as fallback "monthly_installment": 454.17, // optional — fixed monthly installment (€) "history": [ { "date": "2022-05-02", // YYYY-MM-DD "principal": 454.17, // principal paid in this transaction (€) "interest": 47.24, // interest paid in this transaction (€) "cost": 2.50 // bank fees / other costs (€) }, { "date": "2024-01-01", "principal": 454.17, "interest": 52.10, "cost": 2.50, "interest_rate": 4.50 // optional — rate changed; overrides global for projections } ] }

FieldTypeDescription
totalnumberRequired. The original total principal of the loan in €.
interest_ratenumberGlobal annual interest rate as a percentage (e.g. 3.97 = 3.97%). Used as a fallback when no entry-level rate is set.
monthly_installmentnumberFixed monthly installment amount in €. Used as the minimum payment in simulations.
history[].datestringTransaction date in YYYY-MM-DD format. Multiple transactions on the same calendar month are aggregated.
history[].principalnumberPrincipal repaid in this transaction (€). Extra lump-sum payments can be recorded here.
history[].interestnumberInterest charged and paid in this transaction (€). Set to 0 for extra lump-sum principal payments.
history[].costnumberAdditional bank fees or transaction costs (€). Set to 0 if none.
history[].interest_ratenumberOptional. Annual interest rate in effect at the time of this transaction (%). Overrides the global rate for upcoming interest projections. The most recent entry with this field set wins — useful for variable-rate mortgages.
